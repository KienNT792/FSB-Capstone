### S1-06 - RTPTorrent CSV Loader to DuckDB

**Priority:** Critical  
**Estimate:** 6h  
**Primary output:** `data/test_history.db`

Use the existing script at `data/scripts/load_rtp_dataset.py`.

Recommended command after S1-03:

```powershell
python data/scripts/load_rtp_dataset.py `
  --db-path data/test_history.db `
  --rtp-path data/repos/rtp-torrent `
  --auto `
  --force
```

Manual project command:

```powershell
python data/scripts/load_rtp_dataset.py `
  --db-path data/test_history.db `
  --rtp-path data/repos/rtp-torrent `
  --projects apache@sling square@okhttp brettwooldridge@HikariCP `
  --force
```

Current schema implemented by the script:

```sql
CREATE TABLE test_runs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    repo        TEXT NOT NULL,
    job_id      TEXT NOT NULL,
    commit_sha  TEXT,
    test_id     TEXT NOT NULL,
    test_index  INTEGER,
    outcome     TEXT NOT NULL,
    duration_ms REAL,
    timestamp   INTEGER,
    run_count   INTEGER,
    UNIQUE(repo, job_id, test_id)
);

CREATE TABLE file_changes (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    repo        TEXT NOT NULL,
    commit_sha  TEXT NOT NULL,
    file_path   TEXT NOT NULL
);
```

Data mapping:

- `<project>.csv`
  - `travisJobId` -> `job_id`
  - `testName` -> `test_id`
  - `index` -> `test_index`
  - `duration * 1000` -> `duration_ms`
  - `count` -> `run_count`
  - `failures`, `errors`, `skipped`, `count` -> `outcome`
- `tr_all_built_commits.csv`
  - `tr_job_id` -> `job_id`
  - `git_commit_id` or `sha` -> `commit_sha`
- `<project>-patches.csv`
  - `sha` -> `commit_sha`
  - `name` -> `file_path`

Outcome derivation:

- `failures > 0 OR errors > 0` -> `FAIL`
- `skipped > 0 AND skipped == count` -> `SKIPPED`
- otherwise -> `PASS`

Acceptance criteria:

- Loader exits 0 for selected projects.
- Re-running with the same DB is idempotent through `UNIQUE(repo, job_id, test_id)` and `INSERT OR IGNORE`.
- Console output reports inserted `test_runs`, inserted `file_changes`, and unmapped SHA rate.
- `data/test_history.db` and DuckDB WAL files stay uncommitted.

### S1-07 - Ground Truth Validation Notebook

**Priority:** High  
**Estimate:** 5h  
**Primary output:** `notebooks/01_ground_truth_validation.ipynb`

Validate `data/test_history.db` and document dataset quality.

Notebook checks:

- Total `test_runs` rows.
- Total `file_changes` rows.
- Per-project row counts.
- Per-project failure rate.
- Per-project null `commit_sha` rate.
- Top 10 most frequently failing tests per selected project.
- Duration distribution: min, median, p95, max.
- Count of `PASS`, `FAIL`, and `SKIPPED`.
- Cross-check against `<project>-offenders.csv` when feasible.

Acceptance criteria:

- Notebook executes end-to-end from repo root.
- Total loaded `test_runs` >= 10,000.
- Projects with null `commit_sha` rate > 40% are flagged.
- Projects with failure rate < 2% are flagged for replacement.
